Vulnerable mode

Builds SQL by inserting raw user input directly into the query string. That makes the SQL logic editable by the input text.

Example idea: a crafted login value can change the WHERE clause and bypass authentication.

Secure mode

Uses prepared statements with placeholders so the database treats user input as data instead of executable SQL.

This is the pattern you should use in real applications.
